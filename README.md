# 相続アドバイザーPass プライバシーポリシー

制定日: 2026年6月15日

本プライバシーポリシーは、Passmate が提供する iOS アプリ「相続アドバイザーPass」（以下「本アプリ」）における利用者情報の取扱いについて説明するものです。

## 1. 本アプリについて

本アプリは、相続アドバイザー3級の学習を目的とした試験対策アプリです。

本アプリは、銀行業務検定協会、経済法令研究会、CBT-Solutions、その他の試験関係団体が提供または承認するアプリではありません。

本アプリは、個別の相続相談、税務相談、法律相談、遺産分割の助言、金融商品の推奨を行うものではありません。

## 2. 取得しない情報

本アプリは、以下の情報を利用者に入力させる機能を提供していません。

- 氏名
- 住所
- 電話番号
- メールアドレス
- 連絡先
- 正確な位置情報
- マイナンバー
- 財産額
- 相続税額
- 金融口座情報
- 個別の相続相談内容

## 3. 端末内に保存される情報

本アプリは、学習機能を提供するため、以下の情報を利用者の端末内に保存します。

- 回答履歴
- 正誤情報
- 学習進捗
- 苦手問題の記録
- 連続学習日数
- 文字サイズなどの設定
- プレミアム利用状態
- 利用状況に関する集計用イベント

これらの学習情報は、原則として本アプリ内で利用され、利用者を特定する目的では使用しません。

## 4. 広告について

本アプリは、未購入状態の利用者に対して広告を表示するため、Google Mobile Ads SDK を利用します。

広告配信に伴い、Google が広告識別子、端末情報、利用状況などを取得する場合があります。Google による情報の取扱いについては、Google のプライバシーポリシーおよび広告に関する説明をご確認ください。

- Google プライバシーポリシー: https://policies.google.com/privacy
- Google 広告に関する説明: https://policies.google.com/technologies/ads

本アプリでは、広告視聴後に一部の問題を追加で利用できるリワード広告を提供する場合があります。

## 5. 利用状況分析について

本アプリは、機能改善や不具合の把握、利用状況の分析のため、Firebase Analytics を利用します。

Firebase Analytics では、アプリ起動、学習開始、回答、学習完了、購入画面の表示などのイベントを、個人を直接特定しない集計用情報として送信する場合があります。

本アプリは、Firebase Analytics に問題本文、選択肢本文、解説本文、氏名、メールアドレス、個別の相続相談内容、財産額、相続税額、金融口座情報、マイナンバーを送信しません。

本アプリでは、Firebase Analytics を広告識別子（IDFA）を利用しない構成で使用し、App Tracking Transparency の許可を求める機能は実装していません。

- Firebase プライバシーポリシー: https://firebase.google.com/support/privacy

## 6. アプリ内課金について

本アプリは、プレミアム機能を提供するため、Apple のアプリ内課金および RevenueCat を利用します。

購入処理は Apple が提供する仕組みにより行われます。本アプリの開発者が、クレジットカード番号などの決済情報を取得することはありません。

RevenueCat は、購入状態の確認、購入の復元、プレミアム機能の有効化のために利用されます。RevenueCat による情報の取扱いについては、RevenueCat のプライバシーポリシーをご確認ください。

- RevenueCat プライバシーポリシー: https://www.revenuecat.com/privacy/
- Apple プライバシーポリシー: https://www.apple.com/legal/privacy/

## 7. 外部送信について

本アプリでは、広告配信、利用状況分析、購入状態の確認、アプリ内課金の処理に必要な範囲で、Google、Firebase、RevenueCat、Apple のSDKまたはサービスと通信する場合があります。

本アプリは、個別の相続相談内容、財産額、相続税額、金融口座情報、マイナンバーを外部送信する機能を提供していません。

## 8. 第三者提供

本アプリの開発者は、法令に基づく場合を除き、利用者を直接特定する個人情報を第三者に提供しません。

なお、広告配信、利用状況分析、アプリ内課金、購入状態確認に関して、Google、Firebase、Apple、RevenueCat がそれぞれのポリシーに基づき情報を取り扱う場合があります。
